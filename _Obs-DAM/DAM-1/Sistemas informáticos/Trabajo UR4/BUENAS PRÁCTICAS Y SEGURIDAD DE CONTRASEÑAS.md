# 7. BUENAS PRÁCTICAS Y SEGURIDAD DE CONTRASEÑAS

La seguridad de los usuarios locales no solo depende de los permisos NTFS, sino de la robustez de sus credenciales.

## 7.1. Recomendaciones de Seguridad (Referencia INCIBE)

El Instituto Nacional de Ciberseguridad (INCIBE) es la entidad de referencia en España que proporciona recursos y guías para proteger sistemas e identidades. Para establecer contraseñas seguras, las recomendaciones técnicas estándar incluyen:

- **Longitud:** Mínimo de 10-12 caracteres.
    
- **Complejidad:** Combinación de mayúsculas, minúsculas, números y símbolos especiales.
    
- **Imprevisibilidad:** Evitar información personal deducible (fechas de nacimiento, nombres de mascotas) y patrones lógicos de teclado (ej. "qwerty").
    

## 7.2. Gestores de Contraseñas

Para manejar múltiples credenciales complejas sin riesgo de olvido, se emplean los gestores de contraseñas. Son aplicaciones que almacenan las credenciales en una "bóveda" digital cifrada, a la que el usuario accede mediante una única "contraseña maestra".

- **Pros:** Permiten tener contraseñas extremadamente largas, distintas para cada servicio, autocompletan inicios de sesión y generan claves seguras automáticamente.
    
- **Contras:** Si se olvida o se ve comprometida la contraseña maestra, se pierde o se expone el acceso a toda la bóveda.
    
- **Ejemplos de software gratuito:** Bitwarden o KeePass.
