## Modelo OSI

> [!info] OSI = Open Systems Interconnection
> Modelo **teórico** de referencia para la interconexión de sistemas. Compuesto por **7 capas**, cada una con función definida.

Cada nivel se comunica con los adyacentes añadiendo **cabeceras** a los paquetes.

| N.º | Capa | Función |
|-----|------|---------|
| 7 | **Aplicación** | Permite a las aplicaciones acceder a las demás capas |
| 6 | **Presentación** | Cifra y comprime los datos |
| 5 | **Sesión** | Permite establecer más de una sesión |
| 4 | **Transporte** | Asegura y confirma que los datos llegaron a destino |
| 3 | **Red** | Encamina los datos de forma óptima |
| 2 | **Enlace** | Agrupa datos y evita errores en la transmisión |
| 1 | **Física** | Se encarga de la transmisión física |

---

## Modelo TCP/IP

> [!info] TCP/IP
> Modelo de **4 capas** utilizado en redes LAN.
> Nombrado por sus dos protocolos principales: **TCP** y **IP**.

| Capa | Función |
|------|---------|
| **Aplicación** | Datos del usuario, envío y recepción |
| **Transporte** | Divide datos y crea paquetes |
| **Internet** | Envía paquetes y establece la mejor ruta |
| **Acceso a la red** | Envío físico de los paquetes |

---

## Comparación OSI vs TCP/IP

```
TCP/IP              OSI
─────────────────────────────
                  Aplicación  ┐
Aplicación        Presentación├ → Aplicación TCP/IP
                  Sesión      ┘
Transporte        Transporte
Internet          Red
Acceso a la red   Enlace      ┐
                  Físico      ┘ → Acceso a la red TCP/IP
```

> [!tip] Flujo de datos
> En **ambos modelos**: la información va de la capa superior → inferior al enviar, viaja por la red, y en el destino va de inferior → superior.

**En TCP/IP:**
1. Datos salen de la capa de **Aplicación**
2. En **Transporte** → se dividen en **segmentos** (TCP) o **datagramas** (UDP) + cabecera
3. Se añade **cabecera IP** con la dirección destino
4. Viajan como **tramas** por el medio físico

## Protocolos por Capa TCP/IP

| Capa TCP/IP | Protocolos |
|-------------|------------|
| **Aplicación** | DNS, DHCP, HTTP, HTTPS, FTP, TFTP, SMTP, POP, IMAP, LDAP, Telnet, SSH, SMB, NFS, SNMP |
| **Transporte** | TLS/SSL, TCP, UDP, SCTP |
| **Internet** | IP, ARP, RARP, ICMP, IGMP, IPSec |
| **Acceso a la red** | Ethernet, Wi-Fi, PPP |

---

## Protocolos de Transporte

| Protocolo | Descripción |
|-----------|-------------|
| **TCP** | Orientado a conexión, garantiza entrega de datos |
| **UDP** | Sin conexión, envío por datagramas (más rápido, menos seguro) |
| **SCTP** | Combina características de TCP y UDP |
| **TLS/SSL** | Cifrado de información entre aplicación y transporte |

## Protocolos de Internet

| Protocolo | Descripción |
|-----------|-------------|
| **IPv4 / IPv6** | Encaminan datos por su dirección IP |
| **IPSec** | Añade seguridad (cifrado y autenticidad) a IP |
| **ARP** | Resuelve dirección IP → dirección MAC |
| **RARP** | Resuelve dirección MAC → dirección IP |
| **ICMP** | Mensajes de control, comprueba conectividad y latencia (usado por `ping`) |
| **IGMP** | Gestiona multidifusión en redes |

## Protocolos de Acceso a la Red

| Protocolo | Descripción |
|-----------|-------------|
| **Ethernet** | Acceso al medio, difusión, todos los nodos comparten canal |
| **WiFi** | Acceso al medio inalámbrico |
| **PPP / PPPoE / PPPoA** | Conexiones punto a punto |

## Puertos de Red

> [!info] ¿Qué es un puerto?
> Número entero (16 bits, rango **0–65535**) que TCP/IP usa para identificar la **aplicación** a la que enviar/recibir datos.

- **Puerto origen**: asignado dinámicamente
- **Puerto destino**: predeterminado pero configurable
- **Socket** = Dirección IP + número de puerto

### Rangos de puertos

| Rango | Tipo |
|-------|------|
| 0 – 1023 | **Puertos bien conocidos** (reservados para servicios principales) |
| 1024 – 49151 | **Puertos registrados** (aplicaciones de usuario) |
| 49152 – 65535 | **Efímeros / dinámicos / privados** |

### Puertos más importantes

| Puerto | Protocolo | Descripción |
|--------|-----------|-------------|
| 21 | FTP | Transferencia de archivos |
| 22 | SSH / SFTP | Acceso remoto seguro |
| 23 | Telnet | Acceso remoto (inseguro) |
| 25 / 464 | SMTP | Envío de correo |
| 53 | DNS | Resolución de nombres |
| 67/68 | DHCP | Asignación dinámica de IP |
| 69 | TFTP | Transferencia de archivos simple |
| 80 | HTTP | Web |
| 110 | POP3 | Recepción de correo |
| 143 | IMAP | Correo en servidor |
| 389 | LDAP | Directorio activo |
| 443 | HTTPS | Web segura |
| 445 | SMB | Compartir archivos |
| 587 | SMTP seguro | Envío de correo cifrado |
| 636 | LDAPS | Directorio activo seguro |
| 993 | IMAP seguro | Correo en servidor cifrado |
| 995 | POP3 seguro | Recepción de correo cifrado |
| 3389 | RDP | Escritorio remoto Windows |
| 5900 | VNC | Escritorio remoto libre |

## Protocolos de Aplicación Detallados

### Web — HTTP / HTTPS
- **HTTP** → puerto **80**, transfiere archivos HTML
- **HTTPS** → puerto **443**, versión segura con certificado SSL/TLS
- HTTP/2 trabaja con **TCP** | HTTP/3 trabaja con **UDP** (más rápido)

### Transferencia de Archivos — FTP / TFTP
| Protocolo | Puerto | Transporte | Seguridad |
|-----------|--------|------------|-----------|
| FTP | 21 | TCP | No seguro |
| TFTP | 69 | UDP | No seguro, para archivos pequeños |
| FTPS | 21 | TCP | FTP sobre SSL |
| SFTP | 22 | TCP | FTP sobre SSH, cifrado asimétrico |

### Acceso Remoto
| Protocolo | Puerto | Descripción |
|-----------|--------|-------------|
| **Telnet** | 23 | Inseguro, casi en desuso |
| **SSH** | 22 | Seguro, sustituye a Telnet |
| **VNC** | 5900 | Escritorio remoto, software libre |
| **RDP** | 3389 | Escritorio remoto Microsoft |

### Correo Electrónico
| Protocolo | Puerto | Función |
|-----------|--------|---------|
| **SMTP** | 464/25 (587 seguro) | **Enviar** correos |
| **POP3** | 110 (995 seguro) | **Recibir** correos (descarga al equipo) |
| **IMAP** | 143 (993 seguro) | **Recibir** correos (trabaja en el servidor) |

### Otros Protocolos
| Protocolo | Puerto | Función |
|-----------|--------|---------|
| **DNS** | 53 | Traduce nombres de dominio → IP |
| **DHCP** | 67/68 | Asignación dinámica de IPs |
| **SMB** | 445 | Compartir archivos e impresoras |
| **LDAP/LDAPS** | 389/636 | Directorio activo |
| **SNMP** | — | Gestión de la red |

### Copias Remotas
| Protocolo | Descripción |
|-----------|-------------|
| **RCP** | Copia remota en Linux (inseguro) |
| **SCP** | RCP sobre SSH (seguro) |
| **RSYNC** | Copiar y sincronizar carpetas remotas |
