
# Principales modelos de nube y sus características


## Nube pública, privada e híbrida
### Nube pública
La nube pública ofrece servicios a múltiples usuarios a través de internet, facilitando el acceso y la escalabilidad.
### Nube privada
La nube privada esta dedicada exclusivamente a una única organización para mayor seguridad y control.
### Nube híbrida
Los datos se guardan dentro de una nube privada por seguridad y utilizan una parte de nube pública para ciertos servicios públicos.

## Modelo comunitario y caso de uso
### Definición de nube comunitaria
la nube comunitaria es una infraestructura de cloud compartida por varias organizaciones con intereses comunes.
### Beneficios del modelo comunitario
Permite compartir recursos y costes entre diferentes organizaciones, mejorando la eficiencia y la colaboración.
### Ejemplos prácticos de uso
Varias organizaciones gubernamentales y sectoriales utilizan el modelo comunitario para organizar servicios y infraestructuras.

# Servicios en nube: Tipologías y aplicaciones empresariales
## Infraestructura como a servicio (IaaS)
### Recursos informáticos virtualizados
IaaS ofrece servidores, almacenamiento y redes accesibles a través de internet para una infraestructura flexible.
### Gestión escalable y flexible
Permite ajustar fácilmente la capacidad informática según las necesidades sin inversión física directa.
## Plataforma como a servicio (PaaS)
### Entorno de desarrollo 
PaaS proporciona un entorno completo para crear y probar aplicaciones sin gestionar la infraestructura.
### Despliegue de aplicaciones
Facilita el despliegue rápido de aplicaciones, reduciendo el tiempo necesario para llevar productos al mercado.
### Innovación acelerada
Permite a las empresas innovar mas rápido mediante la simplificación de la gestión técnica.
## Software como servicio (SaaS)
### Acceso vía internet
El SaaS permite acceder a las aplicaciones directamente a través de internet sin instalación local.
### Actualización centralizada
Las aplicaciones SaaS se mantienen y actualizan centralmente, simplificando la gestión del software.
### Aplicaciones empresariales comunes
SaaS es ideal para aplicaciones como el correo electrónico y la gestión de clientes en empresas.

# Fog computing y Mist computing: Extensión de la computación distribuida
## Diferencias entre fog, mist i edge computing
### Fog Computing
Fog computing funciona como una capa intermedia, ofreciendo procesamiento y almacenamiento cercanos a la nube y al edge.
### Mist Computing
Mist computing acerca aún más el procesamiento a los dispositivos finales para mejorar la capacidad de respuesta y la eficiencia.
### Comparación con Edge Computing
Edge computing implica el procesamiento directo en los dispositivos finales, mientras que la computación en la niebla y la computación en la bruma añaden capas intermedias para optimizar.
## Casos de uso en sectores industriales
### Fabricación avanzada
La computación fog y mist mejora la supervisión y la automatización en la fabricación para optimizar la producción y la seguridad.
### Sector sanitario
Estas tecnologías permiten una mejor monitorización de los pacientes y aumentan la seguridad en los entornos sanitarios.
### Transporte inteligente
Fog i mist computing optimizan la seguridad y la gestión del transporte con datos distribuidos en tiempo real.
