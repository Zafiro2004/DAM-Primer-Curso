## Definición de ordenador
Un **<font color="#00b050">ordenador</font>** (o computadora) es un **<font color="#00b050">sistema electrónico programable que procesa datos (entradas)</font>**, los **<font color="#00b050">transforma</font>** mediante <font color="#00b050">operaciones aritmético-lógicas y los almacena</font> o <font color="#00b050">presenta como resultados (salidas)</font>. Está <font color="#00b050">compuesto</font> esencialmente por <font color="#00b050">componentes físicos (hardware) y lógicos (software)</font>, que en conjunto permiten realizar tareas diversas bajo el control de instrucciones predefinidas (programas). En su concepción académica, un ordenador requiere al menos:

1. Una <font color="#00b050">unidad central de procesamiento (CPU)</font>, que interpreta y ejecuta instrucciones.
2. Una **<font color="#00b050">memoria</font>**, para almacenar tanto datos como instrucciones: memoria principal de trabajo (p.ej. RAM) y memoria no volátil.
3. Un subsistema de <font color="#00b050">entrada/salida (E/S)</font>, que permite comunicar el sistema con el exterior (usuarios, otros sistemas, dispositivos).

El ordenador <font color="#00b050">debe cumplir</font>, además, <font color="#00b050">con mecanismos de sincronización (buses, relojes internos), control de errores, gestión de recursos y coordinación entre sus componentes internos.</font>
## Definición de Hardware
El <font color="#00b050">hardware</font> se refiere al <font color="#00b050">conjunto de componentes físicos</font> que componen un ordenador o sistema informático, tanto los **<font color="#00b050">internos</font>** (dentro del chasis) como los **<font color="#00b050">periféricos</font>** externos. Incluye elementos mecánicos, electrónicos, electromecánicos y componentes que intervienen en el suministro de energía, procesamiento, almacenamiento, comunicación, etc. 

El <font color="#00b050">hardware trabaja de forma estrecha con el software (y el firmware)</font>, pues sólo adquiere funcionalidad cuando se le dota de instrucciones. Muchos <font color="#00b050">aspectos del rendimiento, capacidad y eficiencia</font> de un sistema **dependen** de las <font color="#00b050">características</font> del **<font color="#00b050">hardware</font>** instalado.
### Tipos de Hardware
Al clasificar el hardware, se pueden distinguir varias categorías según su función, ubicación o grado de importancia para el funcionamiento básico del ordenador.

| Categoria                                           | Componente Físico                                                                                                                      | Función / Funcionamiento                                                                                                                                                                                                                                                                            |
| --------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Procesamiento**                                   | <font color="#00b050">CPU</font> (unidad central de procesamiento), <font color="#00b050">GPU</font> (unidad de procesamiento gráfico) | La <font color="#00b050">CPU ejecuta instrucciones</font> de los programas: hace <font color="#00b050">operaciones aritméticas, lógicas, controla flujo, llamadas a E/S</font>. La <font color="#00b050">GPU procesa datos gráficos</font>, imágenes y vídeo, liberando carga de trabajo de la CPU. |
| **Memoria principal**                               | <font color="#00b050">RAM</font> (Random Access Memory)                                                                                | <font color="#00b050">Memoria volátil</font> que conserva datos e instrucciones mientras esté encendida; permite al <font color="#00b050">CPU acceder rápidamente</font> a lo que se está ejecutando.                                                                                               |
| **Almacenamiento no volátil**                       | Discos duros (HDD), discos de estado sólido (SSD), unidades ópticas, memorias flash                                                    | Permite <font color="#00b050">guardar datos y programas de forma persistente</font> aunque el sistema esté apagado; su velocidad, capacidad y tipo (mecánico, electrónico) influyen en la rapidez del sistema.                                                                                      |
| **Placa base (motherboard)**                        | Placa madre, chipset, buses internos                                                                                                   | <font color="#00b050">Integra y conecta todos los componentes de hardware</font>: CPU, memorias, dispositivos de E/S, proporcionando rutas eléctricas y lógicas para que los datos circulen. Además <font color="#00b050">gestiona la distribución de energía interna.</font>                       |
| **Fuente de alimentación y componentes de soporte** | Fuente de alimentación, ventiladores, radiadores de calor, disipadores, sistemas de refrigeración                                      | <font color="#00b050">Suministran energía eléctrica estable</font>, mantienen t<font color="#00b050">emperaturas adecuadas</font>, evitan fallos físicos por calor o mala alimentación.                                                                                                             |

### Hardware interno vs externo

- **<font color="#00b050">Interno</font>**: componentes que están <font color="#00b050">**dentro del chasis/carcasa**</font> del ordenador; <font color="#00b050">esenciales para que el sistema pueda arrancar y operar</font> (CPU, RAM, placa base, etc.).
- **<font color="#00b050">Externo</font>**: dispositivos <font color="#00b050">conectados a la unidad central</font>, llamados periféricos, que <font color="#00b050">permiten entrada/salida, almacenamiento auxiliar, interacción con el usuario</font>. <font color="#00b050">**No todos los periféricos**</font> son estrictamente <font color="#00b050">necesarios para el funcionamiento</font> mínimo del sistema.
### Definición de periférico
Un **<font color="#00b050">periférico</font>** es <font color="#00b050">cualquier dispositivo que se conecta a un ordenador para expandir su funcionalidad</font> permitiendo la interacción con el usuario o con otros sistemas, o bien para almacenar información auxiliar, pero que <font color="#00b050">no pertenece al núcleo </font>imprescindible del sistema de procesamiento central. En esencia, los <font color="#00b050">periféricos suministran funciones de entrada/salida</font> (entrada al sistema, salida al usuario, almacenamiento externo, comunicación). El ordenador puede funcionar sin muchos de ellos, aunque <font color="#00b050">su ausencia puede limitar su usabilidad.</font>

### Tipos de periféricos
Los periféricos se clasifican según la dirección en que fluye la información con respecto al ordenador, o bien por su función específica (entrada, salida, almacenamiento, comunicaciones). A continuación las categorías principales con ejemplos y funcionamiento.


| Tipo                                       | Función principal                                              | Ejemplos                                                                           | Cómo funciona                                                                                                                                                              |
| ------------------------------------------ | -------------------------------------------------------------- | ---------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Periféricos de entrada**                 | Introducir datos en el sistema                                 | Teclado, ratón, escáner, micrófono, cámara web                                     | Detectan acciones físicas (teclas, movimientos, imágenes, sonido), convierten esas señales en datos digitales que el ordenador pueda procesar.                             |
| **Periféricos de salida**                  | Presentar o devolver resultados al usuario                     | Monitor/pantalla, impresora, altavoces, proyectores                                | Reciben los datos procesados (texto, gráficos, audio) y los convierten en formatos perceptibles para el usuario (imagen, sonido, impresión).                               |
| **Periféricos mixtos / de entrada-salida** | Permiten tanto introducir como extraer o almacenar información | Unidades USB, discos externos, pantallas táctiles, módems, impresoras multifunción | Funcionan bidireccionalmente: por ejemplo, una unidad USB puede recibir archivos desde el ordenador y luego esos archivos pueden leerse y mostrarse desde la misma unidad. |

#### Firmware

El término **firmware** hace referencia a un tipo especial de software estrechamente ligado al hardware, que reside en memorias no volátiles y cuya misión es controlar el funcionamiento interno de los dispositivos electrónicos. Su papel es esencial en la medida en que actúa como intermediario entre el hardware físico y el software de alto nivel, garantizando la inicialización, la configuración básica y la comunicación entre componentes.

El firmware se encuentra habitualmente almacenado en memorias ROM, PROM, EPROM o, más recientemente, en memorias flash reprogramables. Puede venir preinstalado por el fabricante, aunque en muchos casos es actualizable para corregir errores, mejorar la seguridad o añadir nuevas funcionalidades.

Existen distintos tipos de firmware. El de arranque, como el BIOS o el UEFI, se encarga de realizar comprobaciones al encender el ordenador y de preparar el sistema para cargar el sistema operativo. También hay firmware asociado a subsistemas específicos, como tarjetas gráficas, controladores de red o unidades de almacenamiento, que gestionan su correcto funcionamiento. Finalmente, se distingue entre firmware de bajo nivel, más estático y difícilmente modificable, y firmware de alto nivel, más flexible y susceptible de actualizaciones.

La importancia del firmware radica en que un fallo en su ejecución puede comprometer de forma grave el funcionamiento del dispositivo, llegando incluso a inutilizarlo.


| Nivel/tipo                                       | Descripción                                                                                                                                                                                                                                                                                                     |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Firmware de arranque (bootloader, BIOS/UEFI)     | Inicializa el hardware al encender el dispositivo, realiza comprobaciones (POST), prepara el sistema para cargar el sistema operativo.                                                                                                                                                                          |
| Firmware de subsistema                           | Gestiona módulos específicos de hardware: por ejemplo controladores integrados, tarjetas de red, GPU, controladores de almacenamiento, etc.                                                                                                                                                                     |
| Firmware de bajo nivel vs firmware de alto nivel | El de bajo nivel suele ser más fijo, con pocas posibilidades de cambio, instruído directamente en memorias ROM u otras memorias no volátiles no fácilmente regrabables. El de alto nivel puede usar memorias flash o similares que permiten actualizaciones, corrección de errores, adición de funcionalidades. |
