## ¿Qué es la máscara de subred?

> [!info] Máscara de subred
> Valor de **32 bits** (igual que IPv4) que indica qué parte de una dirección IP pertenece a la **red** y qué parte pertenece al **host**.

También sirve para **dividir una red en subredes**.

---

## Notaciones

La máscara se puede expresar de dos formas equivalentes:

| Forma | Ejemplo |
|-------|---------|
| **Decimal con puntos** | `255.255.255.0` |
| **Notación CIDR** | `/24` |

CIDR = **Classless Inter-Domain Routing** (enrutamiento entre dominios sin clases)

### Tabla de máscaras habituales

| Máscara decimal | CIDR | Bits de red | Bits de host | Hosts válidos |
|-----------------|------|-------------|--------------|--------------|
| 255.0.0.0 | /8 | 8 | 24 | 2²⁴ - 2 = 16.777.214 |
| 255.255.0.0 | /16 | 16 | 16 | 2¹⁶ - 2 = 65.534 |
| 255.255.255.0 | /24 | 24 | 8 | 2⁸ - 2 = 254 |
| 255.255.255.128 | /25 | 25 | 7 | 2⁷ - 2 = 126 |
| 255.255.255.192 | /26 | 26 | 6 | 2⁶ - 2 = 62 |
| 255.255.255.224 | /27 | 27 | 5 | 2⁵ - 2 = 30 |
| 255.255.255.240 | /28 | 28 | 4 | 2⁴ - 2 = 14 |
| 255.255.255.248 | /29 | 29 | 3 | 2³ - 2 = 6 |
| 255.255.255.252 | /30 | 30 | 2 | 2² - 2 = 2 |

## Cómo definir una red

Se necesita la **dirección de red** y la **máscara de subred**:

```
Dirección de red:  192.168.0.0
Máscara de subred: 255.255.255.0

Equivalente en CIDR: 192.168.0.0/24
```

---

## Calcular la dirección de red — Operación AND

> [!info] Operación AND
> Para obtener la dirección de red a partir de una IP y su máscara, se realiza una **AND bit a bit**.
> - 1 AND 1 = 1
> - 1 AND 0 = 0
> - 0 AND 0 = 0

### Ejemplos de cálculo de dirección de red

> [!example] Ejemplo 1
> ```text
> IP:       192.168.0.32   →  11000000 . 10101000 . 00000000 . 00100000
> Máscara:  255.255.255.0  →  11111111 . 11111111 . 11111111 . 00000000
> -----------------------------------------------------------------------
> Red:      192.168.0.0    →  11000000 . 10101000 . 00000000 . 00000000
> ```

> [!example] Ejemplo 2
> ```text
> IP:       192.168.56.1   →  11000000 . 10101000 . 00111000 . 00000001
> Máscara:  255.255.255.0  →  11111111 . 11111111 . 11111111 . 00000000
> -----------------------------------------------------------------------
> Red:      192.168.56.0   →  11000000 . 10101000 . 00111000 . 00000000
> ```

---

### Cómo se distribuyen los bits

> [!info] Visualización de una IP con máscara /24
> ```text
>    192     .    168     .     16     .     0
> 11000000       10101000     00010000    00000000
> 
> <------------------- red ------------------> <---host--->
> <----------------- 24 bits ----------------> <--- 8b --->
> <-------------------------- 32 bits -------------------->
> ```

> [!tip] Regla de oro de la Máscara de Subred
> * Los bits de la máscara puestos a **`1`** → identifican la **red**.
> * Los bits de la máscara puestos a **`0`** → identifican el **host**.

---

## Número de hosts válidos

```
Hosts válidos = 2ⁿ - 2
```

Donde **n** = número de bits reservados para host (bits a 0 en la máscara).

Se restan **2** porque:
- Una IP es la **dirección de red** (todos los bits de host = 0)
- Una IP es la **dirección de broadcast** (todos los bits de host = 1)

### Ejemplo con /24

```
Bits de host = 32 - 24 = 8
Hosts válidos = 2⁸ - 2 = 256 - 2 = 254

Red:        192.168.0.0      (todos los bits de host a 0)
Primer host: 192.168.0.1
Último host: 192.168.0.254
Broadcast:  192.168.0.255    (todos los bits de host a 1)
```

## Actividades resueltas

### Actividad 5.4 — Calcular dirección de red
**Datos:** IP = `192.168.56.1`, máscara = `255.255.255.0`

```
  192.168.56.1   → 11000000.10101000.00111000.00000001
AND
  255.255.255.0  → 11111111.11111111.11111111.00000000
= 192.168.56.0   → 11000000.10101000.00111000.00000000
```
**Resultado:** La dirección de red es `192.168.56.0`

---

### Actividad 5.6 — Rango válido y broadcast de 172.16.0.0/16
**Datos:** Red = `172.16.0.0/16`

```
Binario red:     10101100.00010000.00000000.00000000
Binario máscara: 11111111.11111111.00000000.00000000
```

```
Primer host: 10101100.00010000.00000000.00000001 = 172.16.0.1
Último host: 10101100.00010000.11111111.11111110 = 172.16.255.254
Broadcast:   10101100.00010000.11111111.11111111 = 172.16.255.255
```

### Actividad 5.7 — ¿Pertenece una IP a la red 192.168.20.0/22?

**Datos:** Red = `192.168.20.0/22` → máscara = `255.255.252.0`

```
Red:     11000000.10101000.00010100.00000000
Máscara: 11111111.11111111.11111100.00000000
```

Rango válido: `192.168.20.1` – `192.168.23.254`

| IP | AND con máscara | ¿Pertenece? |
|----|-----------------|-------------|
| `192.160.20.5` | → 192.160.20.0 | ❌ No |
| `192.168.22.5` | → 192.168.20.0 | ✅ Sí |
| `192.168.24.5` | → 192.168.24.0 | ❌ No |
