## Configuración en VirtualBox

La configuración de red se puede modificar en **Configuración → Red**.
Se pueden tener hasta **4 adaptadores** por máquina virtual.

### Modos de Red en VirtualBox

| Modo | Descripción |
|------|-------------|
| **NAT** | La VM tiene internet de la máquina anfitriona, pero no se ve con otras VMs |
| **Adaptador puente** | La VM tiene su propia IP en la red real |
| **Red interna** | Red solo entre VMs |
| **Solo-anfitrión** | Red entre la VM y el anfitrión únicamente |
| **Red NAT** | Las VMs se ven entre sí y tienen internet del anfitrión |
| **No conectado** | Sin conexión de red |

## Comandos de Red en Linux

> [!tip] Ayuda de comandos en Linux
> `man <comando>` · `<comando> --help` · `info <comando>`

### `ip` — Gestión de interfaces y rutas

```bash
ip address              # Ver todas las interfaces
ip a                    # Abreviatura
ip address show eth0    # Ver interfaz concreta
sudo ip link set eth0 down   # Desactivar interfaz
sudo ip link set eth0 up     # Activar interfaz
ip neighbour            # Ver tabla ARP
ip route                # Ver rutas de red
```

### `ifconfig` — Configuración de interfaces (clásico)

> Hay que instalar: `sudo apt install net-tools`

```bash
ifconfig           # Mostrar interfaces activas
ifconfig -a        # Mostrar todas las interfaces
ifconfig eth0 up   # Activar interfaz
ifconfig eth0 down # Desactivar interfaz
```

### `iwconfig` — Interfaces inalámbricas

```bash
iwconfig   # Mostrar info de interfaces WiFi
```

### `hostname` — Nombre del equipo

```bash
hostname                    # Ver nombre del equipo
hostname -d                 # Ver nombre + dominio DNS
hostname -i                 # Ver dirección IP del host
hostname -f                 # Ver FQDN completo
```

### `hostnamectl` — Cambiar nombre del equipo (systemd)

```bash
hostnamectl                         # Ver configuración actual
hostnamectl hostname                # Ver nombre del equipo
hostnamectl hostname equipolinux    # Cambiar nombre
hostnamectl set-hostname equipolinux
```

**Tipos de nombre:**

| Tipo | Descripción |
|------|-------------|
| **pretty** | Nombre largo, cualquier carácter |
| **static** | Para el sistema y la red |
| **transient** | Para la red si el static no es válido |

> El nombre se almacena en `/etc/hostname`. Hay que **reiniciar** para que surta efecto.

### `ping` — Comprobar conectividad

```bash
ping localhost              # Ping al propio equipo (infinito, Ctrl+C para parar)
ping -c 4 localhost         # Solo 4 paquetes
ping -c 4 127.0.0.1         # Loopback IPv4
ping -c 4 ::1               # Loopback IPv6
ping -4 google.com          # Forzar IPv4
ping -6 google.com          # Forzar IPv6
ping paraninfo.es
```

> Envía datagramas **ICMP ECHO_REQUEST** y espera **ECHO_RESPONSE**. También mide la latencia.

### `host` — Buscar DNS

```bash
host google.com             # IP del dominio
host 8.8.8.8                # Nombre a partir de IP
```

### `nslookup` — Consultar servidores DNS

```bash
nslookup                    # Modo interactivo (exit para salir)
nslookup google.com         # Resolución de dominio
nslookup paraninfo.es
```

### `traceroute` — Ruta de los paquetes

```bash
traceroute localhost
traceroute paraninfo.es
```

### `netstat` — Conexiones y estadísticas

> Siendo reemplazado por `ss` e `ip`

```bash
netstat -r      # Tablas de enrutamiento
netstat -n      # Mostrar direcciones (no nombres)
netstat -s      # Estadísticas por protocolo
netstat -a      # Todas las conexiones y puertos
netstat -l      # Solo puertos en escucha
```

### `ss` — Ver conexiones del equipo

```bash
ss -a   # Todos los sockets
ss -l   # Solo los que están a la escucha
```

## Servicios de Red en Linux

### NetworkManager
- Usado en entornos de **escritorio**
- Configuración en: `/etc/NetworkManager`

### systemd-networkd
- Usado en entornos de **servidor**
- Configuración en: `/etc/systemd/network`

```bash
# Ver qué servicio está activo
systemctl status NetworkManager systemd-networkd
systemctl is-active NetworkManager systemd-networkd

# Gestionar el servicio
sudo systemctl stop NetworkManager
sudo systemctl start NetworkManager
sudo systemctl enable NetworkManager
sudo systemctl disable NetworkManager
```

### Netplan
- Herramienta de administración de redes de **Ubuntu**
- Configuración en: `/etc/netplan/*.yaml`

```bash
netplan apply   # Aplicar configuración
netplan try     # Probar y revertir si no se confirma
```

## Comandos de Red en Windows

> [!tip] Ayuda en Windows
> `<comando> /?` muestra la ayuda de cada comando.

### `getmac` — Ver dirección MAC

```cmd
getmac
getmac /v        # Salida detallada
```

### `ipconfig` — Ver y gestionar IPs

```cmd
ipconfig         # Ver configuración básica
ipconfig /all    # Ver toda la información
ipconfig /release    # Liberar IP IPv4
ipconfig /renew      # Renovar IP IPv4
ipconfig /release6   # Liberar IP IPv6
ipconfig /renew6     # Renovar IP IPv6
```

### `hostname` — Ver nombre del equipo

```cmd
hostname
```

### `ping` — Comprobar conectividad

```cmd
ping localhost
ping 127.0.0.1
ping ::1
ping -4 google.com   # Forzar IPv4
ping -6 google.com   # Forzar IPv6
ping -t google.com   # Continuo (Ctrl+C para parar)
ping -n 4 google.com # Solo 4 paquetes
```

### `tracert` — Ruta de los paquetes

```cmd
tracert localhost
tracert paraninfo.es
```

### `nslookup` — Consultar DNS

```cmd
nslookup localhost
nslookup google.es
nslookup paraninfo.es
```

### `netstat` — Conexiones y estadísticas

```cmd
netstat -r      # Tablas de enrutamiento
netstat -n      # Mostrar direcciones
netstat -s      # Estadísticas por protocolo
netstat -e      # Estadísticas Ethernet
netstat -a      # Todas las conexiones
netstat -f      # Nombres FQDN externos
netstat -nr     # Combinar -n y -r
```

### PowerShell

```powershell
Get-NetIPAddress             # Ver configuración IP
Test-NetConnection localhost # Probar conexión
```

## Tabla Resumen: Linux vs Windows

| Función | Linux | Windows |
|---------|-------|---------|
| Ver interfaces | `ip address` | `ipconfig /all` |
| Ver MAC | `ip link` / `ifconfig` | `getmac /v` |
| Ver nombre equipo | `hostname` | `hostname` |
| Cambiar nombre | `hostnamectl hostname nombre` | Sistema → Configuración |
| Comprobar conexión | `ping -c 4 host` | `ping host` |
| Ver ruta | `traceroute host` | `tracert host` |
| Ver DNS | `nslookup` / `host` | `nslookup` |
| Ver rutas | `ip route` / `netstat -r` | `netstat -r` |
| Ver puertos | `ss -a` / `netstat -a` | `netstat -a` |
